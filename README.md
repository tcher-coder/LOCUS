# LOCUS — Telegram-агент «ссылка → знание»

Личный Telegram-агент для автоматического создания конспектов из статей, видео (YouTube/TikTok/Vimeo/X) или текста и их интеграции в персональную базу знаний (LLM Wiki по концепции Карпатого).

Проект работает на базе подписки **Claude Pro** через **Claude Agent SDK** без расхода средств на платное API.

---

## 1. Системные требования

На хосте должны быть установлены:
- **Node.js** (v18+)
- **Claude CLI** (Claude Code)
- **Python 3.11+**
- **Git**
- **FFmpeg** (требуется для скачивания/обработки видео)
- **yt-dlp** (требуется для извлечения субтитров и медиа)

---

## 2. Быстрый старт и настройка

### Шаг 1. Клонирование и структура проекта
Проект имеет следующую структуру каталогов:
- `bridge/` — исходный код Telegram-бриджа на Python.
- `vault/` — независимый Git-репозиторий базы знаний (конспекты в `vault/raw/`, вики в `vault/wiki/`).
- `prompts/` — шаблоны промптов для агента.

### Шаг 2. Установка зависимостей
Установите зависимости Python:
```bash
pip install -r bridge/requirements.txt
pip install yt-dlp
```

Установите глобальный скилл видео для Claude Code:
```bash
npx skills add bradautomates/claude-video -g
```

### Шаг 3. Авторизация Claude Pro (OAuth Token)
Запустите в терминале команду:
```bash
claude setup-token
```
Пройдите авторизацию в браузере. Утилита сгенерирует токен, действующий **1 год**.
Скопируйте его и вставьте в файл `bridge/.env` как `CLAUDE_CODE_OAUTH_TOKEN`.
*Дата выпуска токена: 2026-07-13*

### Шаг 4. Настройка Telegram-бота и архивного канала
1. Создайте нового бота у [@BotFather](https://t.me/BotFather) и скопируйте полученный токен в `bridge/.env` как `BOT_TOKEN`.
2. Узнайте свой Chat ID (например, через [@userinfobot](https://t.me/userinfobot)) и вставьте его как `OWNER_CHAT_ID` в `bridge/.env`. Бот будет обрабатывать сообщения только от этого ID.
3. Создайте приватный Telegram-канал для архива, добавьте созданного бота в администраторы канала с правом публикации сообщений. Узнайте ID этого канала (начинающийся на `-100...`, например, переслав любое сообщение из канала боту [@userinfobot]) и вставьте его как `ARCHIVE_CHANNEL_ID` в `bridge/.env`.

### Шаг 5. Запуск Smoke-теста
Для проверки корректности настройки запустите:
```bash
python bridge/smoke.py
```
Все проверки должны завершиться со статусом `[OK]`.

## 3. Использование бота

### Отправка материалов
- **Ссылка на статью**: Просто отправьте боту ссылку. Он скачает страницу через `WebFetch`, создаст конспект в `vault/raw/`, обновит вики и пришлет конспект в чат.
- **Ссылка на видео**: Отправьте ссылку на YouTube/TikTok/Vimeo/X. Бот извлечет субтитры с помощью `/watch` (в режиме `transcript` для экономии лимитов).
  - Чтобы включить распознавание кадров, добавьте к сообщению флаг `+frames` (короткий алиас `+f`; активирует режим `efficient`).
- **Сырой текст**: Отправьте любой текст. Бот сделает из него конспект и обновит связанные концепты в вики.
- **Глубокий анализ**: Добавьте флаг `+deep` (короткий алиас `+d`) к любому сообщению, чтобы запустить агента с повышенным уровнем мышления (`effort="medium"`).

### Управляющие команды
- `/ask <вопрос>` — задать вопрос по базе знаний. Бот найдет информацию в `vault/wiki/` и ответит со ссылками на страницы концептов, не изменяя файлы.
- `/digest` — показать сводку последних изменений из файла `wiki/log.md`.
- `/lint` — запустить интерактивную сессию очистки базы знаний (проверка дубликатов, битых ссылок, устаревшей информации).
- `/stats` — количество обработанных задач за сегодня, размер очереди и оставшийся срок действия OAuth-токена.

### Архивные команды (Выполняются локально без вызова ИИ-агента)
- `/archive [N]` — показать список последних N (по умолчанию 10) документов из `vault/raw/` с инлайн-кнопками для быстрого скачивания.
- `/find <слова>` — быстрый текстовый поиск по содержимому всех `.md`-файлов конспектов. Выводит до 10 совпадений со сниппетами строк и инлайн-кнопками скачивания.
- `/get <slug>` — отправить файл конспекта по названию/части названия. Если совпадений несколько — выведет список вариантов на выбор.
- `/backfill` — разовая ручная синхронизация архива: публикация в канал-архив всех документов из `vault/raw/`, которых там еще нет. Отправка идет с паузой в 3 секунды. Реестр отправленных файлов сохраняется в `bridge/archive_index.json`.

### Диалоговый режим
Если агенту требуется уточнение (например, при краулинге sitemap найдено более 20 страниц, или требуется подтверждение плана), он задаст вопрос в чат. Ответьте на это сообщение бота с помощью функции **Reply (Ответить)**, чтобы продолжить ту же сессию.

## 4. Запуск 24/7 (Развертывание)

### Рекомендуемый способ: Docker (любой Linux-сервер)
В репозитории есть `Dockerfile` (Python + Node/claude CLI + ffmpeg + yt-dlp + скилл `/watch`) и `docker-compose.server.yml` (long polling — порты и reverse-proxy не нужны):

```bash
git clone https://github.com/tcher-coder/LOCUS.git && cd LOCUS
cp bridge/.env.example .env   # заполнить секреты; chmod 600 .env
docker compose -f docker-compose.server.yml up -d --build
docker logs -f locus-bridge
```

При первом старте entrypoint разворачивает `vault/` из `vault.template/` и инициализирует в нём git. База знаний живёт на хосте (bind-mount `./vault`), состояние архива — в `./data`.

Конкретная инсталляция этого инстанса описана в `DEPLOYMENT.md` (не в git).

### Альтернатива: локальный запуск (Windows/Linux)
`python bridge/main.py` + автозапуск средствами ОС (Task Scheduler / systemd). Требования из раздела 1 должны стоять на хосте.
